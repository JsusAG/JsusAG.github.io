Lesson 16: Understanding the Importance of Responsive Web Design

Chapter 16 revolves around RWD which is Responsive Web Design. This is how websites adapt dependomg on factors like device size as well as window size for desktops and laptops. The content will be displayed differently on each device to suit its needs, this is done via the media queries. You may be asking why there is a need for such, but in reality more people browse the web on mobile devices, many which are different sizes. The overall goal of responsive web design is to create the best viewing experience possible to the widest audience.

Before responsive web design the web designers would have to create entirely separate sites for mobile users. As one can imagine this would be a tedious task and technically can cost more for a company. Another major topic in this chapter is Progressive Enhancement. This is the creation of a strong base for the website so that adding enhancements could be easy to implement. The idea behind progressive enhancement is that you could focus on the content being delivered on the site instead of what browser or device people are using. One of the most fun part of websites are the interactivity, which can make or break an experience on a website. The addition of JavaScript you can spice up the website and make it an overall more pleasant experience and more visualy applealing at the same time.

Lesson 17: Designing for Mobile Devices

As the title of the lesson suggests this chapter focuses on designing websites for mobile devices. Sites being designed well on mobile devices can be favorable to websites that are just designed well for desktops since mobile devices outweigh desktops by a significant margin. For example, since more and more individuals are using movile devices having a poorly designed site, or a site that doesn't even work on mobile can lose potential customers. Google even started to use "mobile-friendliness" as a criteria for it's search engine. Here is a list of some of the things Google looks for in a website: font sizes, images ad other box sizes, are they using flash or not, configuring the viepoint, and tappable elements among others.

One of the things to look out for in a mobile site would be flash, since you have to download a flash player to see the flash content not all individuals will have it installed practically making it useless to most individuals. One thing you want to have on the site is tappable links. Not only the ability to tap the links, but also the easability of the tappable links like having them spaced out enough to not cause unwanted tapps to the wrong links. Another point to lookout for when designing for mobile would be simplicity. This can be missed easily since while a desktop has plenty of space the mobile devices dont have that liberty, making it harder to design certain elements for mobile users. Some designers even use the Mobile first design philosophy which has strenghts and weaknesses in design, but does work nonetheless. This involves focusing on getting the required content and functionality to as many users/customers as possible, then you enhance the site for more diverse devices afterwards.

Lesson 18: Using Media Queries and Breakpoints

This chapter opens with the media query, which is a logical expression that is either true or false. The true or false is determined if the media type of the media query matches the media type of the device. When it comes to media types there are ten different ones you can test for with css media queries and they are: all, braille, embossed, handheld, print, projection, screen, speech, tty, and tv. Out of all these the Screen media type is the most common one used today. Media features are features that allow you to look at specific features and let you style your pages according to your specs. Of these media features there exists 13 and they are: aspect-ratio, color, color-index, device-aspect-ratio, device-height, grid, height, monochrome, orientetion, resolution, scan, and width.

When using media queries you need to know how to write expressions and how they work. There are four logical operators that are used to define complex scenarios, they are and, not, comma-separated list, and only. The and operator combines features together, not negates the entire query, and the comma-separated list is the same as the OR logical operator, and finaly the only operator will prevent browsers from applying styles when media queries are not supported. And finally breakpoints, this is a feature with a media query declaration assigned to it. Most web designers think of breakpoints as places where the width of the design changes in order to compensate fot different devices like the diference between an android vs an ios phone. The more breakpoints you have on the site, the more "versions" of the page you would need to style. Now these breakpoints are defined with media queries, and most of the time they are determined by the width of the device. There are three breakpoints you general design for in a mobile design environment. They are the smaller phones and devices, the larger phones, devices and tablets(beween 480px and 1200px), and finally one for larger widths (larger than 1200px).
